First off, thank you for considering contributing to Scene Scout.

To ensure the project remains legally sound and to protect the long-term viability of the software, we require all contributors to agree to a Contributor License Agreement (CLA) before we can accept pull requests.

### How to Sign the CLA
You do not need to sign a physical document. To signify your agreement to this CLA, you must include the following exact statement in the description of your Pull Request:

> **"I have read the `CONTRIBUTING.md` file and agree to the terms of the Scene Scout Contributor License Agreement."**

Pull requests without this declaration will not be reviewed or merged.

## How to Contribute

1. **Fork the Repository:** Create your own fork of the Scene Scout repository.
2. **Create a Branch:** Create a new branch for your feature or bugfix (`git checkout -b feature/your-feature-name`).
3. **Write Clean Code:** Ensure your code matches the existing style of the project. We follow standard Python PEP 8 guidelines.
4. **Test Your Changes:** Make sure your modifications do not break existing functionality.
5. **Submit a Pull Request:** Open a PR against the `main` branch of the original repository. Ensure you include the CLA agreement text in your PR description.

## Reporting Bugs

If you find a bug, please open an issue on the repository. Include:
* Your operating system and hardware specifications.
* The steps to reproduce the bug.
* The expected behavior vs. the actual behavior.
* Any relevant error logs or tracebacks.
